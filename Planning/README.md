# Planning
## Hoofdlijnen
De eerste week zullen we de website ontwikkelen in HTML en CSS zodat we deze kunnen presenteren aan Sander onze projectmanager. De volgende stap zal zijn het ontwikkelen van de website in Umbraco. Eerst door tutorials te kijken en dan het ontwikkelen in Umbraco zelf. Daarna zullen we effectief gaan developpen in c#. De views aanmaken voor de verschillende pagina's en ook de templates in Umbraco definiëren. Hierna zal de website naar acceptatie gebracht worden. Waarop Sander en Bart (.NET developpers) onze website zullen testen. Daarna zal de website getoond worden aan het management en zullen we nog extra feedback ontvangen. Zij zullen waarschijnlijk nog wel bepaalde zaken kunnen opmerken of tips kunnen geven.

## Toelichting fases
De eerste fase zal zijn het opbouwen van de website in HTML en CSS met behulp van de template. In de tweede fase zal er wat tijd voorzien worden zodat we ons kunnen inwerken in Umbraco aangezien dit voor ons een nieuwe technologie is. Tijdens de derde fase zullen we het content-management systeem umbraco effectief gaan integreren in onze website. Dus content toevoegen alsof de website effectief online zou staan. Dit wil zeggen dat de website gehost zal worden op een aparte server waar het management en een paar .NET programeurs testen op zullen uitvoeren. Tijdens de vierde fase gaan we onze website in de acceptatie omgeving brengen. Als vijfde fase zullen we de website van de acceptatie server naar de productie server brengen. Als er nadien nog wijzigingen toegevoegd moeten worden zullen deze wijzigingen getest worden in de acceptatie fase en pas nadien naar de productie fase worden overgebracht. De laatse fase zal het presenteren van de website zijn en de feedback van HR verwerken.
